# RESEARCH-001 — 네이버 블로그 자동화 조사

작성일: 2026-07-17
상태: 조사 완료 / 실제 로그인 테스트 전

## 결론

현재 NAVER Developers에서 생활백서맘 프로그램이 사용할 수 있는 공개형 네이버 블로그 글쓰기 REST API는 확인되지 않았다.

과거 블로그 글쓰기 API 공지는 존재하지만 현재 공식 문서에서 확인되는 기능은 블로그 공유하기와 종료된 블로그 앱 URL Scheme 안내다. 따라서 과거 API를 현재 사용 가능한 발행 API로 간주하지 않는다.

## 가능한 운영 방식

### A. 공식 공유 기능
- 제목과 외부 URL을 네이버 블로그 공유 창으로 전달한다.
- 전체 원고·다중 이미지·서식·발행 상태를 LifeBookMom-OS가 직접 통제하기 어렵다.
- 생활백서맘 원클릭 발행의 주 방식으로는 부적합하다.

### B. 승인형 브라우저 자동화
- 사용자가 네이버에 직접 로그인한다.
- LifeBookMom-OS가 준비된 원고와 이미지로 글쓰기 화면을 채운다.
- 발행 직전 QA를 실행한다.
- 기본 정책은 자동 발행이 아니라 `READY_FOR_REVIEW` 상태에서 멈춘다.
- 실제 발행은 사용자 승인 후 실행한다.

### C. 복사·붙여넣기 보조
- 브라우저 자동화가 불안정하거나 정책 위험이 있으면 제목, 본문, 이미지 순서, 태그를 클립보드 패키지로 제공한다.
- 가장 안정적인 폴백 경로다.

## 정책 안전 기준

1. 네이버 계정 비밀번호를 저장소나 설정 파일에 저장하지 않는다.
2. CAPTCHA, 2단계 인증, 보안 확인을 우회하지 않는다.
3. 로그인 세션은 사용자가 직접 생성한 로컬 브라우저 프로필만 사용한다.
4. 발행 전 사용자 확인 단계를 기본값으로 둔다.
5. 네이버 화면 구조 변경 시 자동 발행을 중단하고 선택자 재검증을 요구한다.
6. 반복 발행 속도 제한과 실패 즉시 중단 규칙을 둔다.

## 다음 실제 검증

- 네이버 글쓰기 화면 DOM 구조 확인
- 제목·본문 입력 가능 여부
- 이미지 1장 업로드 가능 여부
- 임시저장 가능 여부
- 발행 직전 멈춤 가능 여부
- 에디터 변경 감지용 selector health check

## 판정

- 공식 API 자동 발행: 현재 FAIL
- 승인형 브라우저 자동화: 검증 후보
- 복붙 보조 자동화: 구현 가능
- 무인 완전 자동 발행: 실제 테스트 전까지 금지
